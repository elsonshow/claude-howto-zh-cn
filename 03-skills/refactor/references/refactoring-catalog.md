# 重构方法清单（Refactoring Catalog）

这个文件汇总了一些常见重构手法及适用场景。

## 使用方式

1. 先识别 smell
2. 找对应 refactoring
3. 小步实施
4. 每一步都跑测试

## 常见 Refactorings

### 提炼方法（Extract Method）

**适合**：长函数、重复逻辑、需要给概念命名  
**做法**：

1. 抽出代码块
2. 给新方法起清晰名字
3. 处理参数和返回值
4. 用新方法替换原代码
5. 测试

### 内联方法（Inline Method）

**适合**：方法体和名字一样简单，抽象没有价值  
**做法**：

1. 找到所有调用点
2. 直接内联
3. 删除原方法
4. 测试

### 提炼变量（Extract Variable）

**适合**：复杂表达式难懂  
**做法**：

1. 抽出中间变量
2. 用有意义命名
3. 替换原表达式
4. 测试

### 重命名变量 / 函数（Rename Variable / Function）

**适合**：命名表达不清  
**做法**：

1. 明确真正职责
2. 批量替换
3. 测试
